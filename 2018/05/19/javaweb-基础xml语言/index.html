xml基础基础运用

xml解析

xml的解析方式

  1. sax：特点：逐行解析，只能查询。
  2. dom: 特点：一次性将文档加载到内容中，形成dom树。可以对dom树curd操作。

    xml解析技术dom4j

  • 比较常用的解析开发包，hibernate底层采用。
  • 利用dom4j完成查询操作
    • 配置相应的文件
    • 创建核心对象new SAXReader();
    • 将xml文档加载到内存形成一棵树
      Document doc = reader.read(文件的位值)
    • 通过文档获取根节点
      Elemnt root=doc.getRootElement();
    • 通过根节点获取其他的节点的元素
      • 获取所有的子元素
        List list=root.elements()
      • 获取元素的指定属性内容
        String value=root.attributeValue(“属性名”);
      • 获取子标签标签体:遍历list 获取到每一个子元素
        String text=ele.elementText(“子标签名称”)

        复习一下反射的技术

  1. 获取对应的class 对象
    • 方式1：
      • class clazz=Class.formName()
    • 方式2：
      • Class clazz=类名.class;
    • 方式3：
      • Class clazz=对象.getclass();
  2. 通过字节码对象获取实例对象
    • Object clazz.newInstance(); 相当于new了一个对象
  3. 通过class对象获取public方法
    • Method method= clazz.getmethod(“方法名字”，Class…paramType)；例如int.class
  4. 让方法执行
    • method.invoke（object 实例对象，object…参数）；
      a.add(10)相当于 method.invoke(a,10);

xml的约束

  • 作用 ：规定xml中可以出现那些元素及那些属性，以及他们出现的顺序。
  • 约束的分类：
    • DTD约束：strust hiebernate等
    • SCHEMA约束： tomact spring等
  1. DTD约束：
    • 和xml的关联 (一般都会提供好,复制过来即可,有时候连复制都不需要.)
      • 方式1:内部关联
        • 格式:
          <!DOCTYPE 根元素名 [dtd语法]>
      • 方式2:外部关联-系统关联
        • 格式:
          <!DOCTYPE 根元素名 SYSTEM “约束文件的位置”>
        • 例如:
          <!DOCTYPE web-app SYSTEM “web-app_2_3.dtd”>
      • 方式3:外部关联-公共关联
        • 格式:
          <!DOCTYPE 根元素名 PUBLIC “约束文件的名称” “约束文件的位置”>

  2. DTD语法了解

    • 元素:

      • <!Element 元素名称 数据类型|包含内容>
      • 数据类型:
        #PCDATA:普通文本 使用的时候一般用()引起来
        • 包含内容:
          该元素下可以出现那些元素 用()引起来
      • 符号:
          • 出现任意次
        • ? 出现1次或者0次
          • 出现至少1次
        • | 或者
        • () 分组
        • , 顺序

    • 属性:

      • 格式:
        <!ATTLIST 元素名 属性名 属性类型 属性是否必须出现>
      • 属性类型:
        ID:唯一
        CDATA:普通文本
      • 属性是否必须出现
        REQUIRED:必须出现
        IMPLIED:可以不出现

一个xml文档中只能添加一个DTD约束

SCHEMA约束

  • 一个xml文档中可以添加多个schema约束
  • xml和schema的关联
    • 格式：
      <根标签 xmlns=”….”….>
      <根标签 xmlns:别名=”….”….>
    • 通过别名区分不同的约束
  • schema约束本身也是xml文件.
    • spring自身就是用的schema
