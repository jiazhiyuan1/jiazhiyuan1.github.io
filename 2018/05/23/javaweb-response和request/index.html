response

  • 作用： 往浏览器写回东西
  • 操作响应行
    • 格式： 协议/版本 状态码 状态码说明
    • 状态码：
      • 1xx:已发送请求
      • 2xx:已完成响应
      • 3xx:还需要浏览器继续操作 302 重定向，304，读缓存
      • 4xx:用户操作错误 404:用户操作错误，405访问的方法错误
      • 5xx:服务器内部异常
    • 常用方法：
      • setStatus（int 状态码）：针对于1 2 3
      • setError （int状态码）：针对于4xx和5xx

  • 操作响应头：

    • 格式key/value(多个值)
    • 常用方法：
      • setHader（String key，String value）设置字符串响应格式
      • setHader（String key，int value）设置整形的响应头
      • setDateHeader（string key，long value）设置时间的响应头
      • addHader（String key，String value）设置字符串响应格式
      • addHader（String key，int value）设置整形的响应头
      • addDateHeader（string key，long value）设置时间的响应头

  • 重定向

    • 方式1 response.sendRedirect(“/day10/loc2”);
    • 方式2 response.setStatus(302);response.add(“location”,”/day10/loc2”);
  • 定时刷新

          request

  • 作用：获取浏览器发送过来的数据

    操作请求行

  • 操作行
    • 掌握
      • string getMethod（）：获取请求方式
      • string getRemoteAddr():获取IP地址
      • string getContextPath():后去java中的项目名称
    • 了解
      • getRequestURI():获取的是从项目名到参数的内容
      • getRequestURL（）：获取带协议的路径
      • String getQueryString():get请求的所有参数 username=tom&password=123
      • String getProtocol():获取协议和版本

  • 操作头

    • 格式:key/value(value可以是多个值)

      • 常用方法:
        ★String getHeader(String key):通过key获取指定的value (一个)

        了解:

        Enumeration getHeaders(String name) :通过key获取指定的value(多个)
Enumeration getHeaderNames() :获取所有的请求头的名称
int getIntHeader(String key):获取整型的请求头
long getDateHeader(String key):获取时间的请求头

    • 重要的请求头:

      user-agent:浏览器内核 msie firefox chrome
referer:页面从那里来 防盗链

      操作请求头
      格式:key/value(value可以是多个值)
      常用方法:

      ★String getHeader(String key):通过key获取指定的value (一个)

了解:
    Enumeration getHeaders(String name) :通过key获取指定的value(多个)
    Enumeration getHeaderNames() :获取所有的请求头的名称
    int getIntHeader(String key):获取整型的请求头
    long getDateHeader(String key):获取时间的请求头

  • 操作请求参数 ★

    username=tom&password=123&hobby=drink&hobby=sleep
常用方法:
    String getParameter(String key):获取一个值
    String[] getParameterValues(String key):通过一个key获取多个值
    Map<String,String[]> getParameterMap():获取所有的参数名称和值

    username:tom
    ---------values---------
    hobby:[drink, sleep]
    =====map=======
    username::[tom]
    password::[123]
    hobby::[drink, sleep]
//////////////////////
请求的中文乱码:
    对于get请求:参数追加到地址栏,会使用utf-8编码,服务器(tomcat7)接受到请求之后,使用iso-8859-1解码,所以会出现乱码
    对于post请求,参数是放在请求体中,服务器获取请求体的时候使用iso-8859-1解码,也会出现乱码

    通用的方法:
        new String(参数.getBytes("iso-8859-1"),"utf-8");
    针对于post请求来说:只需要将请求流的编码设置成utf-8即可
        request.setCharacterEncoding("utf-8");
