http初识

  • 超文本传输协议： 规定数据的格式
  • 浏览器往服务器发送 —-请求response
  • 服务器往浏览器发送—-响应

request

  • 组成部分
    • 请求行，请求体，请求头
  • 请求行
    • 请求信息的第一部分
    • 格式： 请求方式 访问的资源 协议/版本 例如：Get /libary/1.html http/1.1
    • 请求方式：
      • get和post：get会把参数放在url的后面 post不会。
        • get有参数大小限制，post没有限制
        • get没有请求体，post有请求体，原因：post需要请求体存放传递的参数
  • 请求头：
    • 格式：key/value(value可以是多个值)
    • 常见的请求头：
      • Accept:text/html,image/bmp –支持的数据类型 大类型/小类型 mine类型
      • Accept-Charset: ISO-8859-1 –字符集
      • Accept-Ecoding：gzip –支持压缩
      • Accept-Language:zh-cn –语言环境
      • Host：www.itcast.cn:80 – 访问主机
      • if-Modified-Since： – 缓存文件最后修改时间
      • referer：来自哪个页面，防盗链
      • user-Agent: Mozilla 4.0(compatible;MSIE 5.5;WIndows NT 5.0)
      • Cookie
      • connnection:Keep-Alive –链接状态
  • 掌握的头信息：
    • referer 来自哪个页面
    • user-Agent 浏览器内核
    • cookie
    • if-modified-since 是否曾经修改

    • response

  • 响应：
    - 组成部分：响应行 ，响应头，响应体
    • 响应行
      • 格式：HTTP/1.1 200 OK 协议/版本 状态码 说明
      • 状态码：200 正常响应成功 302重定向 304读缓存 404用户资源不存在 500服务器内部异常
    • 响应头 ：格式 key value
      • 常见的头：
        • location：http：//www.it315.org/index.jsp –跳转方向和302一起使用
        • Content-Encoding: gzip –数据压缩
        • Content-Type：text/html,charset=GB2312 数据类型
        • Last -Modified：
        • refresh： 1；url=http：// –定时刷新
        • Set-Cookie:SS=Q0=5Lb_nQ; path=/search
        • Expires: -1 –缓存
        • Cache-Control: no-cache –缓存
        • Pragma: no-cache –缓存
        • Connection: Keep-Alive –连接

          Last-Modified:response，If-Modified-Since：request

          以前没有关注Http头的Last-Modified 与If-Modified-Since这个信息，今天在看servlet源码时看到这个标识，就认真学习了一下，理解了一下这两个东西的用途。 Last-Modified 与If-Modified-Since都是用来记录页面的最后修改时间。当客户端访问页面时，服务器会将页面最后修改时间通过 Last-Modified 标识由服务器发往客户端，客户端记录修改时间，再次请求本地存在的cache页面时，客户端会通过 If-Modified-Since 头将先前服务器端发过来的最后修改时间戳发送回去，服务器端通过这个时间戳判断客户端的页面是否是最新的，如果不是最新的，则返回新的内容，如果是最新的，则 返回 304 告诉客户端其本地 cache 的页面是最新的，于是客户端就可以直接从本地加载页面了，这样在网络上传输的数据就会大大减少，同时也减轻了服务器的负担。

          servlet探索

  • void init（servletConfig config）:初始化
    • 初始化方法：服务器
    • 执行次数：一次
    • 执行时机：默认第一次访问的时候
  • void service（servletRequest request，servletresponse response）
    • 服务
    • 执行者：服务器
    • 执行次数： 请求一次执行一次
    • 执行时机：请求来的时候
  • void destroy（）：销毁
    • 执行时机：当servlet被移除或者服务器正常关闭的时候
  • servlet是单实例多线程
    • 默认第一次访问的时候，服务器创建servlet，并调用init实现初始化操作，并调用一次sercice方法，每当请求来的时候，服务器创建一个线程，调用service方法来执行自己的业务逻辑。
    • 当servlet被移除或者服务器正常关闭的时候，服务器调用service方法执行自己的业务逻辑。

      ServletConfig

  • servlet的配置对象
    • 作用：
      • 1.获取当前servlet的名字
      • 2.获取当前servlet的初始化参数
      • 3.获取全局管理者
    • 方法：
      • String getServletName(){}
      • String getInitParameter(){}
      • Enumeration getInitParameterNames() :获取所有的参数名称
      • 初始化参数是放在 web.xml文件
        servlet标签下子标签 init-param
        • ★getServletContext():获取全局管理者
          servletconfig是由服务器创建的,在创建servlet的同时也创建了它,通过servlet的init(ServletConfig config)将config对象
          传递给servlet,由servlet的getServletConfig方法获取
